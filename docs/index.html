安全设计

安全不是补丁
是架构原则

每一条安全机制都从设计之初就嵌入架构，而非事后补救。

🔑

AEAD 认证加密

AES-256-GCM，Authenticated Encryption with Associated Data。每条记忆独立 96-bit nonce，密码学安全。

🏺

强密钥派生

PBKDF2-SHA256，100000 次迭代，32 字节输出。盐值随机生成，密钥从不以明文存储。

🔗

机器绑定

密钥文件绑定机器特征哈希。换机器无法直接使用，需重新认证，防止密钥文件泄露。

📋

完整审计日志

每一次读写操作都有记录：操作者、时间戳、隐私级别、访问字段。日志不含任何记忆内容。

🧹

导出脱敏

导出时自动识别并脱敏敏感信息（手机号、邮箱、密码、证件号

☁️

云端上传禁用

架构层面硬编码禁用所有云端上传路径。STRICT 级别存储物理隔离，任何途径都无法绕过。

隐私分级

四级隐私分级
物理隔离

不只是标记，是硬编码的访问控制。每一级都有独立的存储路径和访问策略。

PUBLIC · 公开
任意模块、任意 Agent 均可访问。无限制，适用于公开信息、一般知识分享。
INTERNAL · 内部
仅限同一 Agent 会话内访问。工作记录、项目进展，默认分级。
PRIVATE · 私密
需显式 ACL 授权才能被其他 Agent 访问。个人偏好、私人信息。
STRICT · 严格
物理隔离独立目录存储，需单独 ACL 授权。密码、证件号、财务信息。

# 隐私扫描 — 自动识别敏感信息并升级分级 from ClawMemory.modules import PrivacyEngine pe = PrivacyEngine(storage) result = pe.scan( "我的手机号是 13812345678，密码是 mysecret123" ) # → Sensitive: True # → Types: ['PHONE', 'PASSWORD'] # → Suggested: STRICT (自动升级) # 授予其他 Agent 访问私密记忆 pe.grant_access( memory_id="xxx", to_agent="claude_code", duration_hours=720 # 30天 )
